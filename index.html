The artifact is a text-based adventure game titled "Deadly Zombie Adventure Game", created during the IT 140: Introduction to Scripting course. The game involves navigating through a set of challenges to survive a zombie apocalypse. I selected this artifact for my ePortfolio because it represents my foundational skills in programming and demonstrates my ability to create an interactive application from scratch.

The artifact was improved by:

  • Adding New Rooms and Items: Introduced "Library," "Cafeteria," and "Emergency Exit" rooms, along with new items like "Magic Potion," "Key-card," and "Night Vision Goggles."
  • Enhanced Game Logic: Implemented puzzles that need to be solved to access certain rooms or items and introduced multiple endings based on the items collected and decisions made.

Enhancement Two: Algorithms and Data Structure

The artifact is a “Course planner program” from the CS 300: Data Structures and Algorithms course at Southern New Hampshire University. This program loads course data from a file, organizes it using an unordered map, and allows users to view course information and prerequisites.

The artifact was improved by:

  • Linked List Addition: The primary enhancement was the addition of a linked list data structure, which made the program more efficient in storing and retrieving course information.
  • Search Functionality: A search feature was added to find specific courses, improving the user experience by making it easier to navigate through large datasets.
  • Error Handling: Improved error handling to catch and handle file input/output errors gracefully, ensuring the program does not crash unexpectedly.

Enhancement Three: Databases

The artifact is a Database Analysis Report based on the Companies database created during the DAD 220: Introduction to Structured Database Environments course. The database stores information about various companies, their offices, and employees.

The artifact was improved by:

  • Security Enhancements: Implemented regular updates and access controls to ensure data privacy and security.
  • Data Integrity: Ensured that data within the Companies database is accurate, consistent, and up to date, with regular data audits and verification procedures.
  • Aggregation Pipeline: Implemented an aggregation pipeline to analyze data, showcasing the number of offices by state and allowing for more detailed insights into company operations.
